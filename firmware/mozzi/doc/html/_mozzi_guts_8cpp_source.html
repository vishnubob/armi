00028 // Interrupts:
00029 // Timer2 has highest priority (on a 328), but you can't have a flexible interrupt rate as well as pwm.
00030 // The TimerOne library is more flexible and allows both pwm and interrupts,
00031 // by using ICR1 for interrupt rate and OCR1A (pin 9) (or OCR1B pin 10) for pwm level
00032 // So we use Timer1 for audio.
00033 // Ideally we'd use Timer0 for control because it has lowest priority, but Arduino uses
00034 // it for time and delay functions, so we use Timer 2 for control and hope it won't
00035 // interfere with the audio interrupt.
00036 // TODO: Timer0 is messed up anyway, it seems, so maybe it's better to
00037 // use it for control and leave Timer2 available for other things?
